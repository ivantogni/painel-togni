1
Criar conta no GitHub
O GitHub é onde o painel vai ficar hospedado. A conta é gratuita.
2
min
✅
Já tem conta no GitHub? Pule direto para o Passo 2 clicando em "Próximo".
1
Acesse o GitHub
Abra o link abaixo e clique em Sign up.
↗ Abrir GitHub
2
Preencha o cadastro
E-mail, senha e nome de usuário. Escolha um nome simples — ele vai aparecer na URL do painel.

Exemplo: se você usar institutotogni, a URL será:
https://institutotogni.github.io/painel/
3
Confirme o e-mail
O GitHub envia um código para seu e-mail. Digite o código para ativar a conta.
ℹ
Não precisa instalar nada no computador. Tudo é feito direto no navegador.
2
Criar o repositório
O repositório é a "pasta" onde o painel vai ficar armazenado no GitHub.
2
min
1
Clique em "New repository"
No canto superior direito, clique no + e depois em New repository.
2
Nomeie o repositório
No campo "Repository name", digite exatamente:
painel-togni
3
Configure como público
Marque a opção Public — obrigatório para o GitHub Pages gratuito funcionar.
Configurações obrigatórias:
● Public  ← marcar isso
○ Private

☑ Add a README file  ← marcar isso também
4
Crie o repositório
Role a página até o fim e clique em Create repository.
3
Subir o arquivo do painel
Agora você vai enviar o arquivo HTML para o repositório que acabou de criar.
2
min
1
Abra o repositório que você criou
Você já deve estar nele. A URL deve ser parecida com:
github.com/SEU-USUARIO/painel-togni
2
Clique em "Add file" → "Upload files"
No repositório, clique no botão Add file ▾ e depois em Upload files.
3
Renomeie o arquivo para index.html
⚠ Importante: antes de fazer o upload, renomeie o arquivo no seu computador de:

painel-semanal-togni.html → index.html

Isso faz o GitHub Pages abrir o painel automaticamente quando alguém acessar a URL.
💡
Para renomear: clique com botão direito no arquivo → Renomear → digite index.html.
4
Arraste o arquivo index.html para a área de upload
Arraste o arquivo ou clique em "choose your files" para selecionar.
5
Confirme o envio
Role a página até o fim e clique em Commit changes.
github.com/institutotogni/painel-togni
📁 painel-togni Add file ▾
📄index.html agora mesmo
📄README.md 1 min atrás
✅
Assim deve ficar o repositório depois do upload.
4
Ativar o GitHub Pages
Agora você ativa a hospedagem para o arquivo ficar acessível por URL.
2
min
1
Vá em Settings do repositório
No menu superior do repositório, clique em ⚙ Settings.
2
Encontre a seção "Pages"
No menu lateral esquerdo, role até encontrar Pages (dentro de "Code and automation").
3
Configure a fonte do Pages
Em "Branch", selecione:
Configuração do GitHub Pages:

Source: Deploy from a branch
Branch: main  /  root

Depois clique em Save → Save
4
Aguarde ~1 minuto e copie a URL
O GitHub vai processar e mostrar a URL do seu painel. Ela vai ser parecida com:
Sua URL vai aparecer assim:

🌐 Your site is live at
https://SEU-USUARIO.github.io/painel-togni/
📋
Copie essa URL e compartilhe com o time. É o endereço fixo do painel — funciona em qualquer computador e celular.
✓
Painel no ar!
O Instituto Togni agora tem uma URL fixa. O time acessa pelo link, os dados vão para o Sheets.
∞
sempre
🎉
Problema do file:// resolvido. Agora o painel roda em https:// e o fetch para o Apps Script funciona normalmente. Os dados chegam na planilha.
Como o time vai usar
1
Compartilhe a URL com o time
Mande o link no grupo de operações. Cada área abre no navegador, preenche e clica em 🚀 Enviar para Sheets.
2
Cada computador configura a URL do Apps Script uma vez
Na primeira vez que alguém abrir o painel em um computador novo, clica em ⚙ Configurar Planilha e cola a URL do Apps Script. Fica salvo no navegador.
3
Para atualizar o painel no futuro
Se precisar atualizar o arquivo HTML, basta acessar o repositório no GitHub, clicar no arquivo index.html → ✏ editar → ou fazer novo upload substituindo o arquivo.
